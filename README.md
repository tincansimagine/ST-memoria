# Memoria — 장기기억 서고

SillyTavern용 장기기억 확장입니다. 대화를 턴 단위로 자동 기록해 구조화된 장기기억으로 보관하고,
매 턴 현재 맥락과 관련 있는 기억만 골라 **토큰 예산 안에서** 프롬프트에 주입합니다.

외부 백엔드·별도 프로그램·임베딩 API가 **전혀 필요 없습니다.** 실리태번 안에서 완결됩니다.

## 설치

1. 이 폴더를 `SillyTavern/data/<user>/extensions/` 아래에 복사하거나,
   확장 탭 → "Install extension"에서 저장소 주소로 설치합니다.
2. 확장 탭에서 **Memoria — 장기기억 서고** 패널을 엽니다.
3. (권장) 설정 탭에서 기록용 **연결 프로필** 또는 **커스텀 API**를 지정합니다. 지정하지 않으면 현재 연결된 API를 그대로 사용합니다.
   기록은 백그라운드에서 턴마다 일어나므로 저렴하고 빠른 모델을 권장합니다.

## 작동 방식

```
AI 응답 완료 ──► [사서] 턴을 구조화된 기억으로 정리해 서고에 보관 (백그라운드, 채팅을 막지 않음)
                  · 기억 10종: 사건/관계/사실/취향/약속/목표/물건/장소/비밀/인상
                  · 인물 도감(프로필·관계), 캐논(세계 설정 키-값), 상태 보드(슬롯별 최신값), 서약
                  · 인용문은 원문의 실제 부분 문자열일 때만 저장 (환각 증거 차단)
                  · N턴마다 "지금까지의 이야기" 자동 요약 → 오래되면 연대기로 병합

사용자 입력 ──► [회상] 로컬 해시 임베딩 + 어휘 일치 + 중요도 + 최신성으로 채점
                  · MMR 다양화, 종류/턴별 상한, 다중 질의(최근 메시지) 보정
                  · 오래된 기억은 "화석화"되어 더 강한 관련성이 있어야 소환됨
             ──► [주입] 토큰 예산에 맞춰 단계적 축소 후 시스템 메시지로 주입
                  · 서약(비밀 유지·미해결 떡밥 등)은 마지막까지 유지
```

### 비밀과 서브텍스트 (지식 경계)

기억에는 공개(`public`) / 비공개(`private`) / 비밀(`secret`) 등급과 소유자가 있습니다.
현재 캐릭터가 소유한 비공개 기억은 **"서브텍스트 전용 — 직접 발설 금지"** 지시와 함께 주입되고,
권한이 없는 기억은 개수만 알려 추측을 금지합니다. 비밀을 아는 캐릭터는 의미심장하게 행동할 수 있지만,
모르는 캐릭터가 갑자기 아는 것처럼 구는 누출을 막습니다.

### 일관성 보장

- **스와이프/리롤**: 해당 턴의 파생 기억이 자동 무효화되고 새 응답으로 재기록됩니다.
- **메시지 편집**: 편집된 턴을 다시 기록합니다.
- **메시지 삭제**: 저장된 턴을 실제 채팅과 대조해 사라진 턴의 기억을 정리합니다.
- 기억은 **채팅별로** 저장됩니다(`chat_metadata`). 채팅 파일과 함께 이동/백업됩니다.

## 주요 기능

| 기능 | 설명 |
|---|---|
| 자동 기록 | 턴마다 백그라운드로 구조화 기억 추출 (끄고 수동 기록만 쓸 수도 있음) |
| **서고에 질문** | "우리 처음 만난 게 언제였지?" — 저장된 기억만 근거로 턴 번호를 인용해 답변 (`/memoria-ask`) |
| **스토리 바이블** | 인물·캐논·서약·타임라인·요약을 한 편의 마크다운 문서로 내보내기 |
| 기억 브라우저 | 검색·종류 필터·편집·삭제·비활성화·**고정**(매 턴 반드시 주입) |
| **인물 도감** | 등장인물의 프로필(역할·나이·직업·외모·특성)과 **인물별 관계**를 자동 추적·편집 |
| 세계 탭 | 캐논(세계 설정) / 상태 보드 / 서약 관리 |
| **현재 장면 한 줄** | 장소·날짜·시각을 자동 추적해 세계 탭 상단과 주입에 한 줄로 표시 (`Scene Now`) |
| 요약 | N턴마다 자동 요약(이전 요약 참조로 일관성 유지), 오래된 요약은 연대기로 병합, 수동 요약/편집 가능 |
| **턴별 기록** | 사서가 남긴 턴 단위 기록을 요약 탭에서 확인·수정 — 고치면 다음 요약부터 반영 |
| **이벤트 연표** | 고백·폭로·죽음 등 이야기의 결정적 전환점만 골라 시간순 연표로 자동 기록 |
| **아이템 도감** | 의미 있는 선물·전개상 중요한 물건의 소유자·상태를 자동 추적 |
| **배경 참고** | 카드·페르소나·월드인포(로어북)를 사서·요약의 배경 자료로 제공 (선택) |
| **자동 숨김** | 요약이 커버한 옛 메시지를 채팅에서 숨겨 컨텍스트 토큰 대폭 절약 (해제 가능) |
| **인계** | 다른 채팅의 요약·고정 기억·인물·캐논·서약을 새 채팅으로 이어받기 (A→B→C 누적) |
| **요약 언어** | 자동/한국어/영어/일본어/하이브리드(영어 서술 + 대사 원문 유지) |
| 과거 채팅 색인 | 미기록 턴 일괄 기록, **범위 지정**(#시작~#끝)과 중단, **실패 턴 자동 재시도** |
| 장면 연출 (실험적) | 생성 직전 연출 LLM이 장면의 목표·피해야 할 전개를 계획해 함께 주입 |
| **커스텀 API** | 사서 전용으로 OpenAI 호환 API(URL/키/모델) 직접 지정 + 연결 테스트 |
| 백업/복원 | 채팅 기억 전체를 JSON으로 내보내기/가져오기 |
| 프롬프트 편집 | 사서/연출/요약/연대기 프롬프트 자유 수정 + 기본값 복원 |

## 슬래시 명령어 · 매크로

- `/memoria` — 켜기/끄기 토글 (`/memoria on`, `/memoria off`)
- `/memoria-remember 내용` — 수동 기억 추가 (자동 정리에서 제외)
- `/memoria-search 검색어` — 기억 검색
- `/memoria-ask 질문` — 저장된 기억만 근거로 스토리 질문에 답변
- `/memoria-index` — 현재 채팅 전체 색인
- `{{memoria_story}}` — 지금까지의 이야기 요약 전체
- `{{memoria_count}}` — 저장된 기억 수

## 설정 가이드

| 설정 | 기본값 | 설명 |
|---|---|---|
| 주입 토큰 예산 | 4000 | 주입되는 장기기억의 최대 토큰. 초과 시 인용→회상→오래된 요약→상태 순으로 축소 |
| 주입 깊이 | 1 | 채팅 내 주입 위치 (0 = 마지막 메시지 바로 앞) |
| 회상 개수 | 10 | 턴당 회상할 기억 수 |
| 요약 주기 | 8턴 | 자동 요약 간격 |
| 요약 언어 | 자동 | 요약·기억이 기록될 언어. 하이브리드는 영어 서술 + 대사 원문 유지 |
| 이전 요약 참조 수 | 3 | 새 요약 생성 시 참조할 이전 요약 수 (0 = 안 함, -1 = 전체) |
| 카드·페르소나 참고 | 켜짐 | 사서·요약이 캐릭터 카드·페르소나 요지를 배경 자료로 참고 |
| 월드인포 참고 | 꺼짐 | 활성 로어북을 배경 자료로 참고 (로어북이 크면 토큰 증가) |
| 자동 숨김 | 꺼짐 | 요약이 커버한 옛 메시지를 숨김. 최근 메시지 보존 수(기본 6)만큼은 항상 표시 |
| 최대 기억 수 | 400 | 초과 시 중요도 낮은 오래된 기억부터 정리 (고정/수동 제외) |
| 사서 최대 토큰 | 30000 | 사서 응답 상한. 요약·기록이 중간에 잘리지 않도록 넉넉하게 |
| API 소스 | 실리태번 | 사서가 쓸 API. 실리태번(현재 API/연결 프로필) 또는 커스텀 OpenAI 호환 API |

## 팁

- **기존 채팅에 도입**: 도구 탭 → "색인 시작"으로 지난 대화를 한 번에 기억화하세요. 범위(#시작~#끝)를 지정할 수도 있습니다.
- **토큰이 부족하면**: 설정 탭의 "요약된 메시지 자동 숨김"을 켜세요. 요약이 커버한 옛 메시지가 컨텍스트에서 빠지고, 그 내용은 "지금까지의 이야기"와 회상으로 대체됩니다.
- **새 채팅방/봇카드 교체 시**: 이전 채팅에서 "내보내기" → 새 채팅에서 "인계 파일 가져오기"를 하면 이야기·기억이 그대로 이어집니다. 인계는 계속 누적됩니다.
- **설정이 헷갈릴 때**: 개요 탭의 "서고에 질문"으로 스토리 사실 관계를 바로 확인하세요. 기억에 없는 건 없다고 답합니다.
- **중요한 설정을 잊게 하고 싶지 않다면**: 기억 브라우저에서 📌 고정하거나 `/memoria-remember`를 쓰세요.
- **떡밥 관리**: 세계 탭의 서약에서 해결된 떡밥은 ✔로 처리하면 주입에서 빠집니다.
- **기록 실패가 떴다면**: 상태줄에 ⚠ 표시가 보이면 도구 탭에서 색인을 다시 실행하면 실패한 턴만 재시도합니다.

## v0.5.0

- **배경 참고**: 캐릭터 카드·페르소나(기본 켜짐), 월드인포/로어북(선택)을 사서·요약의 배경 자료로 제공 — 이름·설정 오해 감소
- **이전 요약 참조**: 새 요약을 만들 때 이전 요약을 참조해 시간·장소·관계 표기의 일관성 유지 (참조 개수 설정: 0 / N / 전체)
- **추적 도감 확장**: 인물 도감에 더해 **이벤트 연표**(결정적 전환점만 시간순 기록)와 **아이템 도감**(중요한 물건의 소유자·상태 추적) 추가 — 켜면 매 턴 주입에 포함되고 인계·스토리 바이블에도 실림
- **현재 장면 한 줄**: 상태 보드에 쌓이는 장소·날짜·시각을 골라 세계 탭 상단과 주입 맨 앞에 한 줄로 정리 (`Scene Now — place · date · time`)
- **턴별 기록 뷰**: 요약 탭에서 턴 단위 기록을 확인·수정 — 수정하면 다음 요약부터 반영
- 사서 프롬프트 전면 재작성 ("다섯 개의 서가" 구조) — 서고 컨셉으로 완전히 재구성
